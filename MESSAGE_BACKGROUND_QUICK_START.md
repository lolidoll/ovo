# 消息页面背景图功能 - 快速开始指南

## 功能简介
用户现在可以在"个性装扮"中自定义消息页面的背景图，包括消息页面、顶部栏和底部栏。背景图会被自动保存，刷新页面后仍然有效。

## 如何使用

### 1. 打开消息背景设置
```
侧边栏 → 个性装扮 → 消息背景
```

### 2. 上传背景图
- **方式一**: 点击上传区域，选择本地图片文件
- **方式二**: 拖拽图片到上传区域
- 支持格式：JPG、PNG、GIF
- 最大大小：10MB

### 3. 应用背景图
- 在"已上传的背景图"列表中找到要应用的背景
- 点击"应用"按钮
- 背景图会立即显示在消息页面上

### 4. 配置应用范围（可选）
勾选以下选项使背景图显示在相应区域：
- ☑ 消息页面（默认已勾选）
- ☐ 顶部栏（聊天导航栏）
- ☐ 底部栏（输入区和工具栏）

### 5. 管理背景图
- **切换背景**: 点击其他背景的"应用"按钮
- **删除背景**: 点击"删除"按钮移除不需要的背景图
- **清除所有**: 点击"清除所有背景"按钮恢复默认外观

## 技术细节

### 存储方式
- 背景图数据存储在 IndexedDB 中
- 所有数据保存在本地浏览器，不上传到服务器
- 清除浏览器数据会导致背景图丢失

### 性能优化
- 使用固定背景（fixed attachment），滚动时保持位置
- 在移动设备上自动优化以保证流畅性
- 后台图片使用 Base64 编码存储

### 兼容性
- ✅ Chrome/Chromium（推荐）
- ✅ Firefox
- ✅ Safari（包括iOS）
- ✅ Edge
- ⚠️ IE11 不支持

## 常见问题

### Q: 上传的背景图在哪里存储？
A: 存储在浏览器的 IndexedDB 本地存储中，不会上传到服务器。

### Q: 如果清除浏览器数据会怎样？
A: 所有背景图数据会被删除，需要重新上传。建议在清除数据前导出背景图。

### Q: 可以上传多少张背景图？
A: 理论上无限制，但受限于浏览器 IndexedDB 的存储空间（通常为 50MB-250MB）。

### Q: 背景图显示不清楚怎么办？
A: 
1. 尝试使用更高分辨率的图片
2. 调整图片的亮度和对比度
3. 选择与应用配色更搭的背景图

### Q: 如何在多个设备间同步背景图？
A: 目前不支持云同步。如需在其他设备使用，需要重新上传或通过其他方式转移。

## 最佳实践

### 背景图选择建议
- **分辨率**: 1920x1080 或以上
- **文件大小**: 500KB-2MB 之间效果最佳
- **颜色**: 选择不会与消息内容冲突的颜色
- **对比度**: 避免选择太深的背景，会影响文字可读性

### 优化技巧
1. 使用半透明或模糊的背景，提高消息可读性
2. 为顶部栏和底部栏选择与消息页面相同风格的背景
3. 定期更换背景保持新鲜感
4. 根据不同场景选择不同风格的背景

## 开发者信息

### API 使用
如果需要在代码中使用消息背景功能：

```javascript
// 获取所有背景图
const backgrounds = MessageBackgroundManager.getBackgrounds();

// 获取当前背景ID
const currentId = MessageBackgroundManager.currentBackgroundId;

// 应用指定背景
const bg = MessageBackgroundManager.getBackground(bgId);
MessageBackgroundManager.applyBackground(bg);

// 清除背景
MessageBackgroundManager.clearBackground();

// 打开UI
MessageBackgroundManagerUI.open();
```

### IndexedDB 数据库信息
- **数据库名**: MessageBackgroundManagerDB
- **版本**: 1
- **对象存储**:
  - `backgrounds` - 存储所有背景图数据
  - `settings` - 存储应用设置（如当前选中背景）
